# openwrt-immichframe-remote
OpenWRT utility to turn on/off one or more digital photo displays running immichframe android app based on the presence of one or more devices connected to the wifi network.

## How it works
This utility requires <code>hostapd</code> running with control interface(s) enabled. This is the default for OpenWrt. It listens to events generated by <code>hostapd</code> (specifically <code>AP-STA-CONNECTED</code> and <code>AP-STA-DISCONNECTED</code>) to determine if it should send the <code>dim/undim</code> command to one or more configured immichframe android apps. 

## Run the utility automatically when the system boots
To ensure this utility runs automatically when OpenWrt boots, you must install the Procd init script. Procd is OpenWrt's native process management system; it ensures hostapd_cli starts on boot and automatically restarts it if it crashes.

### Step 1: Copy the Init Script to the /etc/init.d/ directory
```
bash
cp ./etc/init.d/immichframe_remote /etc/init.d/
```

### Step 2: Set Permissions and Enable on Boot
Run these three commands in your OpenWrt terminal to make the script executable, link it to the system boot sequence, and start it immediately:
```
bash
# 1. Make the init script executable
chmod +x /etc/init.d/immichframe_remote

# 2. Enable it to run automatically on every system boot
/etc/init.d/immichframe_remote enable

# 3. Start the service right now without rebooting
/etc/init.d/immichframe_remote start
```

### Step 3: Verify It Is Running
You can verify that procd has successfully launched your process by checking the running processes:
```
bash
ps | grep hostapd_cli
```
You should see an output line matching /usr/sbin/hostapd_cli -i wlan0 -a /usr/bin/immichframe_remote.
